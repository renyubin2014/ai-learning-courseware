学习目标

学会用评估样本、指标和回归测试判断 AI 功能是否真的变好。

学习路径

按右侧步骤学习。遇到不熟的概念，进入“关键词学习”集中点开。

1
先定义对象区分模型评估、任务评估和产品评估。
2
建立样本用 Golden Set 覆盖真实和边界场景。
3
拆开指标质量、成本、延迟、安全要分开看。
4
持续回归每次 prompt、模型或知识库变化都要重测。

先学什么：评估不是排行榜

AI 产品评估的第一性原理：你不是在证明某个模型“最强”，而是在证明它能否在你的任务、数据、成本和风险边界内稳定工作。

模型评估

看模型基础能力，例如推理、摘要、代码、多模态理解。它能给方向，但不等于你的产品可用。

任务评估

看模型在具体任务上的表现，例如门店风险判断、客服摘要、制度问答。

产品评估

看用户是否完成任务、是否减少时间、是否减少错误、成本是否可接受。

核心句
评估要回答：这个 AI 功能在真实任务里是否更可靠、更省钱、更有价值。

只看一次演示或通用 benchmark，无法支撑产品决策。

评估对象：先拆清楚你到底在测什么

很多评估混乱，是因为把模型、prompt、RAG、工具、界面和人工流程混在一起看。先拆开，才能知道该改哪里。

模型

同一批样本下，不同模型的正确率、稳定性、成本和延迟如何。

Prompt

同一模型下，不同任务说明、示例、输出格式是否改善结果。

RAG

正确资料是否被召回，模型是否忠实使用资料并给出引用。

工作流

用户是否能理解、编辑、确认、回写结果，错误是否能被发现。

离线评估：先用 Golden Set 打底

上线前先准备一批固定样本，覆盖常见任务和边界情况。它不需要很大，但必须代表真实失败。

收样本 从真实问题、历史工单、模拟任务中选。
标答案 人工写出可接受答案和拒答条件。
跑版本 同一批样本跑不同 prompt 或模型。
打分 按 rubric 或字段级标准评分。
记录失败 把失败归因到模型、资料、prompt 或流程。

样本不要只选简单题

必须包含资料不足、信息冲突、长上下文、噪音输入、高风险问题。

答案不一定只有一个

生成式任务可以用评分标准，而不是追求完全一样的文本。

在线实验：看真实用户是否受益

离线评估通过，只说明可以试。真正的产品价值要看用户行为：是否采纳、是否少改、是否更快、是否愿意复用。

A/B Test

把用户或任务随机分到不同版本，看真实指标差异。

人工修改率

AI 输出后用户改了多少，能直接反映可用程度。

任务完成率

用户是否真的完成了选址分析、客服处理、报告生成等目标。

成本与延迟

质量提升如果带来过高成本或等待时间，也可能不值得上线。

质量保障：防止今天能用，明天退化

AI 系统会因为模型版本、prompt、知识库、业务规则、用户输入变化而退化。质量保障的重点是持续回归。

Regression

每次改 prompt、模型、RAG 或工具，都用固定样本重跑，检查旧能力有没有坏。

Data Drift

用户问题、业务规则和资料分布变化后，旧评估集可能不再代表真实世界。

人工抽检

关键场景保留人工 review，持续发现新失败模式。

红线指标

事实错误率、安全违规、权限泄露、错误动作率必须有硬阈值。

查看答案

Golden Set 应该覆盖哪些样本？

参考答案

应覆盖高频任务、复杂任务、资料不足、信息冲突、长上下文、噪音输入和高风险问题。

查看答案

离线评估和在线实验有什么区别？

参考答案

离线评估看输出是否达标；在线实验看真实用户是否更快、更准、更愿意采纳，以及成本是否可接受。
